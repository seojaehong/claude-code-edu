## 2. 우리가 나은 점 (Site A의 강점)

### 2-1. 다직군 범용성

Site A는 노무사, 변호사, 회계사, 세무사, 의사 등 5개 직군의 CLAUDE.md 템플릿을 제공한다 (3-2강). Site B는 노무사만 다루므로 변호사가 수강하면 모든 예시를 자기 업무에 번역해야 하는 인지적 부담이 발생한다.

**구체적 우위:**
- 3-2강에서 5개 직군별 완성된 CLAUDE.md 템플릿 제공
- 2-1강 문서 분석 예시가 "계약서/취업규칙/법령" 등 범용적
- 수강 대상 시장 규모: 노무사 ~3,000명 vs 전문직 전체 ~200,000명+

### 2-2. UI/UX 컴포넌트 시스템

Site A는 6가지 전용 Callout 컴포넌트를 체계적으로 사용한다:

| 컴포넌트 | 용도 | Site B 대응 |
|----------|------|------------|
| `TryThis` | 실습 유도 (파란색) | 없음 (텍스트로만 안내) |
| `Tip` | 팁/보충 (노란색) | 유사하지만 비정형 |
| `Warning` | 주의사항 (빨간색) | 유사 |
| `KeyPoint` | 핵심 정리 | "핵심 정리" 섹션 |
| `BeforeAfter` | 전후 비교 (그리드) | 텍스트 기반 비교 |
| `CodeBlock` | 구문 강조+복사 (Shiki) | 기본 code block |

특히 `BeforeAfter` 컴포넌트는 시나리오/Before/After/결과를 시각적 그리드로 보여줘서 학습 효과가 높다. `TryThis`는 매 레슨 끝에 배치되어 능동적 학습을 유도한다.

### 2-3. 시간 효율성

Site A는 6.5시간에 핵심 내용을 압축했다. 전문직 종사자는 바쁘다. 46시간짜리 과정을 완주할 확률 vs 6.5시간 과정을 완주할 확률은 비교할 수 없다.

- **완주율 추정**: 6.5시간 과정 70-80% vs 46시간 과정 15-25%
- **ROI**: Phase 1-2만 완료해도 (2시간) 즉시 업무 적용 가능
- **점수/시간 비율**: A 5.2점/hr vs B 0.83점/hr (6.3배 차이)

### 2-4. 모바일 UX

Site A는 `MobileBottomNav`, `MobileSidebar`, `TableOfContents` 등 모바일 전용 컴포넌트가 구현되어 있다. 출퇴근 중 스마트폰으로 학습할 수 있는 환경.

## 3. 레퍼런스가 나은 점 (Site B의 강점)

### 3-1. 콘텐츠 볼륨 및 깊이 (47 vs 23 레슨)

Site B의 47개 레슨은 단순히 양만 많은 것이 아니라, 각 주제를 훨씬 깊이 파고든다.

| 주제 | Site A | Site B |
|------|--------|--------|
| 4대보험 계산 | 2-3강에서 검증 프롬프트 1개 | 3-1강에서 Python 계산기 전체 코드, 검증 체크리스트, 복잡 케이스(교대근무) |
| 스킬/커맨드 | 4강 4개 레슨 (개념→제작→실전→공유) | 5강 4개 레슨 + 파이프라인 체이닝까지 |
| MCP | 5강 4개 레슨 | 8강 4개 레슨 (동일) + 6강 플러그인 + 7강 마켓플레이스 |
| 고급 주제 | 6강 (비대화형, Desktop, Remote, 통합) | 9강 Web dev + 10강 Worktree + 11강 Remote/팀 워크플로우 |

### 3-2. 도메인 특화의 깊이

Site B는 노무사 실무에 깊이 침투한다:

- **퇴직금 계산기** (3-1): 실제 Python 코드로 평균임금→퇴직금 계산. 상여금 포함 여부, 교대근무 야간수당 등 복잡 케이스 처리
- **취업규칙 자동 검토** (3-2): 근로기준법 각 조문 대조 자동화
- **진정서/구제신청서 작성** (3-3): 사실관계→법리→결론 구조의 자동 생성
- **판례 검색 자동화** (3-4): 대법원 판례 API 연동 시나리오
- **실제 보험요율**: 국민연금 4.5%, 건강 3.545%, 장기요양 12.81%, 고용 0.9% 등 2024년 기준 수치

### 3-3. IT 개념의 법률 비유 설명

Site B는 비개발자를 위해 IT 개념을 노무법 비유로 설명한다:

| IT 개념 | Site B의 비유 |
|---------|-------------|
| HTML/CSS/JS | 사건 문서의 내용/서식/자동기능 |
| Git | 파일 변경 이력 추적 시스템 (판례 변경 이력처럼) |
| Worktree | 여러 사건 서류를 별도 책상에 동시 펼쳐놓기 |
| Branch | 원본 파일 건드리지 않는 별도 작업 흐름 |
| Repository | 모든 파일과 변경 이력이 담긴 프로젝트 작업 공간 |

Site A는 이런 비유 전략이 부족하다. "터미널은 텍스트로 컴퓨터와 대화하는 창" 정도의 간단한 설명만 있다.

### 3-4. 고급 주제 커버리지

Site B만 다루는 주제:

| Phase | 주제 | 실무 가치 |
|-------|------|----------|
| 7 | 마켓플레이스 (공식/커뮤니티/MCP 레지스트리) | 도구 생태계 이해 |
| 9 | HTML/CSS/JS 기초 → Next.js → Vercel 배포 | 나만의 웹 도구 제작 |
| 10 | Git Worktree 병렬 AI 코딩 | 2-5배 생산성 향상 |
| 11-3 | 팀 워크플로우 통합 | 사무실 전체 AI 전환 |

특히 Phase 9 (웹 개발)은 "노무 상담 신청 폼"을 직접 만들어 Vercel에 배포하는 프로젝트를 포함한다. 비개발자가 자신의 웹 도구를 만든다는 경험은 매우 강력한 학습 동기가 된다.

### 3-5. 일관된 레슨 구조

Site B의 모든 47개 레슨이 동일한 구조를 따른다:

```
학습목표 → 본문 (3-5섹션) → Before/After → 코드 예시 → 핵심 정리 → 네비게이션
```

Site A도 유사한 구조를 갖추고 있지만, 학습목표가 사이드바/메타데이터에만 있고 본문 상단에 명시적으로 표시되지 않는 점이 차이다.

## 5. 개선 로드맵

### 즉시 가능 (Quick Wins) — 1-3일

| # | 개선 항목 | 예상 소요 | 기대 효과 | 참고 |
|---|----------|----------|----------|------|

## 6. 결론

### Site A의 포지셔닝

Site A는 **"바쁜 전문직을 위한 6.5시간 실전 속성 과정"**이다. 핵심 가치는 시간 효율성과 다직군 범용성. 수강생이 Phase 1-2만 완료해도 (2시간) 당장 업무에 Claude Code를 적용할 수 있다.

### Site B의 포지셔닝

Site B는 **"노무사를 위한 46시간 종합 마스터 과정"**이다. 핵심 가치는 도메인 깊이와 고급 주제 커버리지. 완주하면 웹 개발, 병렬 작업, 팀 자동화까지 가능하지만, 완주율이 낮을 위험이 있다.

### 경쟁 전략

Site A가 Site B를 이기려면 Site B를 따라 47개 레슨으로 확장할 필요가 없다. 대신:

1. **Quick Wins (1-3일)**: 샘플 파일, 에러 FAQ, 학습목표 표시 → 즉각적 품질 향상
2. **차별화 유지**: 다직군 범용성과 시간 효율성은 Site A만의 강점. 이를 더 강화
3. **선택적 깊이 추가**: Worktree와 팀 워크플로우만 2-3개 레슨으로 추가 (Site B의 강점을 압축 흡수)
4. **시각 자료 투자**: 스크린샷/GIF 10-15개만 추가해도 체감 품질이 크게 상승

**최종 평가**: Site A는 34.1점으로 Site B(38.0점)에 3.9점 뒤지지만, Quick Wins만 실행해도 37-38점 수준에 도달할 수 있다. 시간 효율성(6.3배)은 이미 압도적이며, 이것이 바쁜 전문직 시장에서의 진짜 경쟁력이다.
