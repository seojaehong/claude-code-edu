## B. 정보의 정확성 (Accuracy)

### Site A (우리): 3.5 / 5

**문제점:**
1. **설치 명령어가 부정확할 가능성:** `irm https://claude.ai/install.ps1 | iex`와 `curl -fsSL https://claude.ai/install.sh | bash`는 공식 설치 방법이지만, Node.js 기반 설치(`npm install -g @anthropic-ai/claude-code`)와의 관계 설명이 불명확하다(1-2강).
2. **4대보험 요율이 2024년 기준:** CLAUDE.md 템플릿(3-1, 3-2강)에 "2024년" 기준 요율이 기재되어 있으나, 현재 2026년이므로 outdated. 장기요양보험 12.81%는 이미 변경되었을 가능성 높음.
3. **MCP 명령어 정확성 의문:** `claude mcp add gmail -- npx -y @anthropic/gmail-mcp` 같은 패키지명이 실제 존재하는 공식 패키지인지 불확실(5-2강).
4. **`--allowedTools` 문법:** 1-4강에서 `claude --allowedTools "Edit,Write,Bash"` 형태를 안내하지만, 실제 Claude Code의 허용 도구 설정 방법과 일치하는지 검증 필요.
5. **HWP 읽기 기능:** 2-1강에서 ".hwp(한글)은 물론이고" 라고 기재했으나, Claude Code의 HWP 파일 직접 읽기 지원 여부는 확실하지 않음.

### Site B (레퍼런스): 4.0 / 5

**강점:** "2026년 최저임금 시급: 10,030원"(4-1강), "2026년 4대보험 요율 기준으로" 같은 연도 명시가 정확하다. 또한 "AI는 때때로 존재하지 않는 판례 번호를 만들어내거나(할루시네이션)"(1-1강)라는 경고가 정직하고 정확하다.

**문제점:** MCP 설정의 JSON 구조 예시(8-1강)에서 패키지명(`@anthropic/mcp-gmail` 등)이 실제 공식 패키지인지 불확실. 스킬 경로가 `.claude/commands/`(5-1강)로 되어 있어 Site A의 `.claude/skills/`와 다른데, 어느 쪽이 최신인지 확인 필요.

### 개선 제안
- **긴급:** Site A의 4대보험 요율을 2026년 기준으로 업데이트
- 설치 명령어와 MCP 패키지명을 공식 문서 기준으로 재검증
- HWP 파일 지원 여부를 명확히 하거나 "일부 형식은 변환이 필요할 수 있습니다" 단서 추가

## D. 교수법 효과성 (Pedagogical Effectiveness)

### Site A (우리): 4.0 / 5

**강점:** 모든 강의가 일관된 구조를 따른다: 개념 설명 → 실전 예시 → BeforeAfter → KeyPoint → Warning → TryThis. 이 반복 구조가 학습자에게 예측 가능한 학습 경험을 제공한다. TryThis 섹션이 "지금 바로 따라해 보세요"로 시작하는 실습 유도가 효과적이다.

**약점:** Phase 간 난이도 곡선이 다소 급하다. Phase 2(문서 분석)에서 Phase 3(CLAUDE.md)로 넘어갈 때 난이도가 급격히 오르지는 않지만, Phase 5(MCP)에서 갑자기 기술적 개념이 등장한다. 사전 개념 도입이 부족하다.

### Site B (레퍼런스): 4.5 / 5

**강점:** 11 Phase 구조로 학습 곡선이 더 완만하다. Phase 1(AI 소개) → Phase 2(터미널 기초) → Phase 3(퇴직금 계산 실습) 순서로 "왜 필요한지 동기부여 → 도구 기초 → 즉각적 실무 적용"의 흐름이 자연스럽다. 특히 Phase 1(1-1강)에서 "노무사의 하루: Before vs After" 시나리오로 동기부여를 강력하게 시작한다.

**구체적 예시:** "과장이 아닙니다. 이 과정에서 핵심은 AI에게 '정확하게 지시하는 능력'입니다"(1-1강) — 학습 동기를 자극하면서도 과대광고를 피하는 절묘한 표현.

### 개선 제안
- Site A의 Phase 4(스킬)와 Phase 5(MCP) 사이에 "왜 이것이 필요한지"를 더 강하게 동기부여하는 도입부 추가

## G. 프롬프트 엔지니어링 품질 (Prompt Quality)

### Site A (우리): 3.5 / 5

**강점:** 프롬프트 예시가 풍부하고 다양하다. 2-1강에서 "취업규칙_A사.docx를 근로기준법 기준으로 검토해줘. 특히 다음 항목을 중점적으로 봐줘: 1. 연차휴가... 2. 해고절차..."와 같이 구조화된 프롬프트를 제시한다.

**약점:**
1. 프롬프트 **작성 원칙**을 가르치지 않는다. 왜 이 프롬프트가 좋은지, 어떤 원리가 적용되었는지 설명이 없다.
2. 프롬프트가 **실패하는 경우**를 다루지 않는다. "이렇게 요청했더니 결과가 이상하면 이렇게 수정하세요" 같은 반복/개선 과정이 없다.
3. 대부분 "복사해서 쓰세요" 수준이지, 학습자가 **스스로 프롬프트를 설계하는 능력**을 키우지 못한다.

### Site B (레퍼런스): 4.0 / 5

**강점:** 프롬프트 예시가 더 구조적이고 실무적이다. 퇴직금 계산 요청(3-1강)에서 "[직원 정보]", "[최근 3개월 급여]", "[상여금]", "[연차수당]" 섹션으로 나누어 입력하는 방식은 프롬프트 구조화의 좋은 예시다. "계산 과정을 단계별로 출력하고, 최종 결과를 보기 좋게 정리해 줘. 결과를 파일로도 저장해 줘"라는 지시도 실용적이다.

**약점:** Site A와 마찬가지로 프롬프트 원리 교육은 부족하다. "왜 섹션을 나누면 좋은지", "왜 출력 형식을 지정하면 좋은지" 등의 메타 설명이 없다.

### 개선 제안
- **프롬프트 원칙 강의 추가:** "좋은 프롬프트의 5가지 원칙" (구체성, 구조화, 출력형식 지정, 맥락 제공, 제약조건 명시)
- **Bad → Good 프롬프트 비교:** "급여 확인해줘" vs 구조화된 버전의 결과 차이 시연
- **프롬프트 반복 개선 시나리오:** 첫 시도 → 결과 확인 → 프롬프트 수정 → 개선된 결과

## H. 학습 후 독립성 (Post-learning Independence)

### Site A (우리): 3.5 / 5

**강점:** 스킬(Phase 4)과 CLAUDE.md(Phase 3)를 가르쳐서 반복 업무 자동화 능력을 제공한다. 6-4강의 "단계별 구축 가이드"(Week 1~4)가 학습 이후의 로드맵을 제시한다.

**약점:** "원리"보다 "레시피"에 치중한다. 스킬 파일 5개를 "복사해서 쓰세요"라고 주지만, 학습자가 새로운 스킬을 **스스로 설계하는 사고방식**을 충분히 가르치지 않는다. "내 업무에서 어떤 패턴을 찾아 스킬로 만들 것인가?"라는 사고 프레임워크가 부족하다.

### Site B (레퍼런스): 4.0 / 5

**강점:** "AI가 계산한 결과를 그대로 의뢰인에게 제출하면 안 됩니다. 반드시 검증 과정을 거쳐야 합니다"(3-1강)와 같은 검증 습관을 심어준다. 검증 체크리스트(5가지 항목)는 학습자가 독립적으로 품질을 관리할 수 있게 한다. "이런 판단은 노무사가 하고, 계산은 AI가 하는 것이 이상적인 역할 분담입니다"(3-1강)라는 원칙이 명확하다.

**약점:** 코딩 관련 강의(Phase 9 HTML/CSS/JS, Phase 10 Worktree)는 비개발자에게 독립적 활용까지 이르기 어려운 수준이다.

### 개선 제안
- "스킬 설계 프레임워크" 추가: "반복 패턴 식별 → 입력/출력 정의 → 절차 작성 → 테스트 → 개선"의 사고 과정을 가르치기
- "AI 결과 검증 체크리스트"를 별도 강의로 추가 (Site B의 3-1강 검증 체크리스트 참고)
- "문제 해결 사고방식" 섹션: "원하는 결과가 안 나올 때 어떻게 접근할 것인가"

## 핵심 분석: Site B가 우위인 근본 원인

Site B는 **노무사 한 직군에 올인**한 전략이 모든 차원에서 품질을 끌어올린다. Site A는 다직군 포괄 전략으로 넓지만 얕다. 이것이 F(전문직 맥락)에서 1.5점 격차로 나타나며, 다른 차원에서도 0.5점씩 영향을 미친다.

- Phase 6(자동화)의 비대화형 모드, 스케줄, 원격 접속 내용은 Site B에도 있지만 Site A가 더 실용적으로 정리

## Top 5 실행 가능한 개선 사항

### 1. 전문직 Pain Point 깊이 강화 (우선순위: 최상)
**현재 문제:** 직군별 pain point가 나열 수준 (표면적)
**개선 방법:** 각 직군별로 1개의 "진짜 고통스러운 시나리오"를 추가. 예: 노무사의 "네트제 역산", 세무사의 "다수 사업자 부가세 신고 마감일 전날", 변호사의 "상대방 준비서면 마감 D-1 판례 검색". 이런 시나리오에서 Claude Code가 어떻게 도움되는지를 before/after로 제시.
**영향:** F차원 3.5→4.5 예상

### 2. AI 결과 검증 체크리스트 강의 추가 (우선순위: 상)
**현재 문제:** "초안입니다, 전문가 검토 필요합니다"라는 경고만 있고 구체적 검증 방법이 없음
**개선 방법:** 별도 강의 또는 기존 강의에 "검증 체크리스트" 섹션 추가. Site B의 퇴직금 검증 5가지 항목처럼, "4대보험 계산 결과 검증 3단계", "의견서 초안 검토 4가지 포인트" 등을 구체적으로 제시.
**영향:** B차원 3.5→4.0, H차원 3.5→4.0 예상

### 3. 프롬프트 원리 교육 강화 (우선순위: 상)
**현재 문제:** 프롬프트를 "복사해서 쓰세요"로 제시하지만, 원리를 가르치지 않아 응용력 부족
**개선 방법:** Phase 2에 "효과적인 프롬프트 작성법" 강의 추가. Bad/Good 비교, 구조화 원칙, 반복 개선 프로세스 포함. "이 프롬프트가 왜 좋은지" 메타 설명을 기존 예시에 추가.
**영향:** G차원 3.5→4.5, H차원 3.5→4.0 예상

### 4. 4대보험 요율/설치 명령어 등 기술 정보 업데이트 (우선순위: 상)
**현재 문제:** 2024년 기준 정보가 다수 포함되어 있어 2026년 현재 기준과 괴리
**개선 방법:** 모든 연도 의존 정보를 2026년으로 업데이트. 설치 명령어, MCP 패키지명을 공식 문서와 대조 검증. HWP 파일 지원 범위를 정확하게 재기술.
**영향:** B차원 3.5→4.5 예상

### 5. 실무 시나리오 복잡도 1단계 상향 (우선순위: 중)
**현재 문제:** 시나리오가 "취업규칙 검토", "급여대장 오류 찾기" 수준에 머물러 있음
**개선 방법:** 각 Phase에 1개의 "복합 시나리오"를 추가. 예: "A사 퇴직 직원이 부당해고를 주장하는 상황에서 ① 취업규칙의 해고 절차 검토 ② 퇴직금 계산 ③ 구제신청서 초안 작성을 연속으로 수행"하는 스토리 라인. 이를 통해 Phase 간 연결성도 강화.
**영향:** C차원 4.0→4.5, D차원 4.0→4.5 예상

## 결론

Site A는 비개발자를 위한 접근성과 UI 구조에서 강점이 있으며, 다직군 포괄이라는 전략적 차별점을 보유한다. 그러나 콘텐츠 **깊이와 현실성**에서 Site B에 비해 전반적으로 0.5~1.5점의 격차가 있다. 위 5가지 개선 사항을 순차 적용하면, 평균 점수를 3.81에서 4.3 이상으로 끌어올릴 수 있으며, Site B와의 격차를 거의 해소할 수 있을 것으로 판단된다.

가장 임팩트가 큰 개선은 **전문직 pain point 깊이 강화**(#1)와 **프롬프트 원리 교육**(#3)이다. 이 두 가지만으로도 콘텐츠 품질 인식이 크게 달라질 수 있다.
